Project strengths

What this system does well

Performance
3.59× scan speedup
Multi-threaded iterative DFS with LIFO thread pool; recursive fallback under saturation keeps memory working set small.
Portability
Unix-like embedded deployment
Pure C++14 with only two dependencies: SQLite (single-file DB) and POSIX APIs, theoretically suitable for Unix-like embedded environments, though tested only on Linux.
Observability
Observable by design
Async non-blocking logger with bounded-queue fallback policy. Atomic metrics and live system state polling via REST API.
Architecture
Correctness under concurrency
Designed FSM to enforce explicit, mutually exclusive scan lifecycle transitions under concurrency.
Architecture
Scan decoupled from DB I/O
Bounded MPSC queue buffers export jobs. Scan workers proceed without waiting for DB writes to complete.
Architecture
Persistent index and scan diffs
SQLite-backed index persists scan results and exposes added, modified, and deleted file changes via REST API.
Reliability
Backpressure without stalling
Bounded queues reject excess request load with HTTP 429. Zero timeouts across 26,000+ requests under sustained overload.
Reliability
No data loss on shutdown
Designed drain mode and graceful shutdown to preserve observability, and drain the export pipeline before exit.
Reliability
Failure-mode benchmarks
Designed burst, overload, and cancel scenarios were each designed to stress a different part of the system and resources.
